# `agentlb` Specification (Go)

Version: `v0.4`
Date: `2026-02-19`

## 1) Purpose

`agentlb` manages multiple isolated Codex sessions by assigning each alias its own `CODEX_HOME` under `~/.agentlb`.

It should:
- Keep auth/config/state isolated per alias.
- Launch Codex (or another configured command) inside selected alias environment.
- Support explicit alias usage and auto-pick usage.
- Auto-pick with persistent round-robin across existing aliases.

## 2) Scope

### In scope
- Command name: `agentlb`
- Config file: `~/.agentlb/config.toml`
- Default run command configurable (default: `codex`)
- Per-run command override flag
- Session create/login command: `agentlb new <alias>`
- Auto-pick command: `agentlb new` (no alias)
- Also support `agentlb` (no subcommand) as auto-pick run
- Deterministic repeat command: `agentlb last`
- Config bootstrap command: `agentlb config init`
- Round-robin alias selection

### Out of scope (v0.4)
- Usage-based scoring and balancing
- `/status` probing for selection
- Remote sync / shared state
- GUI/TUI session picker

## 3) CLI Contract

### 3.1 Primary Commands

1. `agentlb`
- Auto-select alias using round-robin.
- Execute configured default command in that alias.

2. `agentlb new <alias>`
- If alias does not exist: create session dir, run login flow, then execute default command.
- If alias exists: reuse and execute default command.

3. `agentlb new`
- Auto-select existing alias using round-robin and execute default command.
- If no managed sessions exist, return actionable error.

4. `agentlb config init`
- Write `~/.agentlb/config.toml` with all default values.
- Overwrite existing config.
- Exit `0` and print config path.

5. `agentlb last`
- Execute run command using the most recently selected alias.
- If no previous alias exists (or it no longer exists), return actionable error.

### 3.2 Flags

Global flags (supported on `agentlb`, `agentlb new`, `agentlb new <alias>`, and `agentlb last`):

- `--cmd "<command string>"`
- `--login-cmd "<command string>"`
- `-- <args...>` pass-through args appended to run command.

### 3.3 Exit Codes

- `0`: success
- `1`: generic runtime/config/state error
- `2`: invalid CLI usage
- `3`: alias not found (reserved)
- `4`: no sessions available for auto-pick
- `5`: login command failed during session creation

## 4) Configuration

Config path: `~/.agentlb/config.toml`

Defaults are used when config is missing. `agentlb config init` writes full defaults.

```toml
[runner]
default_command = "codex"
default_command_args = []
login_command = "codex login"

[sessions]
alias_pattern = "^[A-Za-z0-9][A-Za-z0-9._-]{0,63}$"
assignment_history_window = 30
```

## 5) Filesystem Layout

```text
~/.agentlb/
  config.toml
  state/
    global.json
    sessions/
      <alias>.json
  sessions/
    <alias>/
  locks/
    state.lock
```

Permissions:
- `~/.agentlb`, `sessions/`, alias dirs, and `state/`: `0700`
- state json files: `0600`
- lock file: `0600`

## 6) Session Isolation Model

For every subprocess launch:
- `CODEX_HOME=~/.agentlb/sessions/<alias>`
- `HOME` remains unchanged.

## 7) Operational Flows

### 7.1 `agentlb new <alias>`
1. Validate alias.
2. Acquire state lock.
3. Ensure alias session directory exists.
4. If newly created, release lock and run login command with alias `CODEX_HOME`.
5. Reacquire lock and persist assignment metadata.
6. Execute run command in alias environment.

### 7.2 `agentlb` / `agentlb new` auto-pick
1. Acquire state lock.
2. Enumerate aliases from `~/.agentlb/sessions/*` (lexicographic order).
3. If none: exit `4`.
4. Pick alias at `round_robin_index % len(aliases)`.
5. Increment and persist `round_robin_index`.
6. Persist assignment metadata.
7. Execute run command in selected alias environment.

### 7.3 `agentlb config init`
1. Ensure `~/.agentlb` layout exists.
2. Write full default config file.
3. Print config path.
4. Exit `0`.

### 7.4 `agentlb last`
1. Acquire state lock.
2. Enumerate aliases from `~/.agentlb/sessions/*`.
3. If none: exit `4`.
4. Load `last_alias` from `state/global.json`.
5. If missing or alias no longer exists: return actionable error.
6. Persist assignment metadata for `last_alias`.
7. Execute run command in `last_alias` environment.

## 8) State Model

`state/global.json`:
- `round_robin_index`
- `last_alias`

`state/sessions/<alias>.json`:
- `alias`
- `created_at`
- `last_used_at`
- `recent_assignments` (bounded list)

## 9) Logging

Default: errors only.
Optional debug env var: `AGENTLB_DEBUG=1`.

## 10) Security

- Never print secrets/auth content.
- Keep all `.agentlb` files private permissions.
- Sanitize alias to prevent traversal.
- Use lock file for concurrent safety.

## 11) Acceptance Criteria

1. `agentlb new a1` creates `~/.agentlb/sessions/a1`, runs login there, then runs command there.
2. `agentlb` and `agentlb new` pick aliases by round-robin order and persist position.
3. `agentlb last` reuses the most recently selected alias.
4. Running `agentlb` never writes managed auth into `~/.codex`.
5. `--cmd` overrides default command for that invocation only.
6. `agentlb config init` writes full default config file.
7. Concurrent invocations do not corrupt state files.
